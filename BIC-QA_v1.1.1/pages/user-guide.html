📖 产品介绍

关于 BIC-QA

BIC-QA（Business Intelligence & Knowledge Query Assistant） 是一款专为企业级数据库知识管理而设计的智能问答系统。作为新一代数据库知识检索解决方案，BIC-QA 致力于为数据库管理员、开发人员和运维工程师提供高效、精准的知识查询服务。

核心定位

BIC-QA 是一个全面的数据库知识问答平台，支持所有主流数据库系统，特别针对国产数据库生态进行了深度优化。系统集成了丰富的数据库知识库，涵盖 SQL 语法、性能优化、故障诊断、最佳实践等全方位内容，为用户提供即问即答的专业数据库技术支持。

支持的数据库

国际主流数据库

  • Oracle
  • MySQL
  • PostgreSQL
  • SQL Server
  • MongoDB
  • Redis

国产数据库

  • 达梦数据库（DM）
  • 人大金仓（KingBase）
  • OceanBase
  • GoldenDB
  • 华为 GaussDB
  • 腾讯 TDSQL

云原生数据库

  • 阿里云 PolarDB
  • 腾讯云 TDSQL-C
  • 华为云 GaussDB

技术优势

  • 智能语义理解：基于先进的 NLP 技术，准确理解用户查询意图
  • 知识图谱构建：构建完整的数据库知识关联网络，提供上下文相关的答案
  • 实时知识更新：持续更新数据库版本特性和最佳实践
  • 多维度检索：支持按数据库类型、问题类型、技术领域等多维度精准检索

🚀 安装指南

系统要求

浏览器支持

  • Chrome 88+
  • Edge 88+
  • 其他基于 Chromium 的浏览器

网络要求

  • 支持 HTTPS/HTTP 请求
  • 稳定的网络连接
  • 防火墙允许扩展访问

安装步骤

步骤 1：下载扩展文件

从官方网站或 GitHub 下载 BIC-QA 扩展压缩包（BIC-QA_v1.1.1.zip）

步骤 2：解压文件

将下载的压缩包解压到本地目录，确保解压后的文件夹结构完整

步骤 3：打开浏览器扩展管理页面

  • Chrome：在地址栏输入 chrome://extensions/ 或通过菜单：更多工具 → 扩展程序
  • Edge：在地址栏输入 edge://extensions/ 或通过菜单：扩展

步骤 4：启用开发者模式

在扩展管理页面的右上角，打开"开发者模式"开关

步骤 5：加载扩展

点击"加载已解压的扩展程序"按钮，选择解压后的 BIC-QA_v1.1.1 文件夹

⚠️ 重要提示：选择的文件夹必须包含 manifest.json 文件。请确保选择的是最内层的插件文件夹，而不是外层压缩包文件夹。

步骤 6：验证安装

安装成功后，浏览器工具栏会显示 BIC-QA 图标。点击图标即可打开插件界面。

⚙️ 配置指南

首次使用 BIC-QA 需要进行四步配置，完成后即可开始使用。

步骤 1：用户注册

操作步骤

  1. 点击浏览器工具栏中的 BIC-QA 图标
  2. 点击右上角的"⚙️ 设置"按钮
  3. 在设置页面找到"用户注册"部分
  4. 填写以下信息：
    • 用户名：您的用户名（必填）
    • 所属企业名称：您所在的企业名称（必填）
    • 邮箱：您的邮箱地址（必填）
  6. 勾选同意用户协议和隐私政策
  7. 点击"注册"按钮
  8. 查收邮件获取 API Key
⚠️ 重要提醒：请勿随意修改默认的注册服务 URL，以免影响插件正常运行。如已修改，请重置设置恢复默认值。

步骤 2：知识库服务配置

操作步骤

  1. 在设置页面找到"知识库服务配置"部分
  2. 将注册邮件中收到的 API Key 填入"API密钥"字段
  3. 确认"服务URL"为默认值（通常无需修改）
  4. 点击"保存配置"按钮
  5. 可点击"测试连接"验证配置是否正确

步骤 3：AI 模型配置

3.1 配置服务商接口

操作步骤

  1. 在设置页面找到"大模型接口配置"部分
  2. 点击"+ 添加服务商"按钮
  3. 填写服务商信息：
    • 服务商类型：选择或自定义服务商类型（如 ollama、deepseek 等）
    • 服务商名称：输入服务商名称（如 OpenAI、百度文心、阿里通义等）
    • API地址：填写 API 接口地址
    • API密钥：填写 API Key（如果需要）
    • 认证类型：选择认证方式（Bearer Token、API Key 或自定义）
    • 请求格式：选择请求格式（OpenAI格式、Claude格式或自定义格式）
  4. 点击"保存服务商"按钮
💡 本地 Ollama 配置示例： 
API 地址：http://localhost:11434/v1
认证类型：Bearer Token（通常本地 Ollama 不需要 API Key）

3.2 自动发现模型

操作步骤

  1. 在服务商配置中，点击"测试"按钮连接服务
  2. 系统会自动发现该服务商下可用的模型
  3. 在模型列表中勾选需要使用的模型
  4. 点击"批量纳管"保存模型配置
💡 提示：首页下拉菜单会显示"模型名（服务商）"格式，避免同名模型混淆。

3.3 手动配置模型

操作步骤

对于不支持自动发现的服务商，可以手动添加模型：

  1. 在设置页面找到"模型配置"部分
  2. 点击"+ 添加模型"按钮
  3. 填写模型信息：
    • 所属服务商：选择已配置的服务商
    • 模型名称：输入模型名称（如 gpt-3.5-turbo、gpt-4 等）
    • 显示名称：输入显示名称（可选）
    • 最大Token数：设置模型的最大 Token 限制
    • 温度参数：设置温度值（0-2，控制回答的创造性）
    • 设为默认模型：勾选此项可将此模型设为默认模型
  5. 点击"保存模型"按钮

步骤 4：参数规则配置（可选）

操作步骤

BIC-QA 支持自定义提示词和检索参数，提供三种预设模式：

模式 温度 相似度 适用场景
创意模式 高 中等 需要创造性回答
精确模式 低 高 需要准确性优先
自定义模式 可调 可调 完全自定义需求

在设置页面的"知识库参数规则配置"部分，您可以：

  • 添加自定义规则
  • 设置相似度阈值（0-1，数值越高匹配越精确）
  • 设置 TOP N（1-10，返回最相关的N个结果）
  • 设置温度参数（0-2，控制回答的创造性）
  • 自定义提示词模板
⚠️ 参数影响：温度越高创造性越强，相似度/TOP N 影响知识库检索范围和数量。

🎯 使用说明

使用模式

弹窗模式

点击浏览器工具栏中的 BIC-QA 图标，会弹出一个小窗口，提供基本的问答功能。适合快速提问和简单对话。

完整页面模式

在弹窗中点击右上角的"🖥️ 完整页面"按钮，会在新标签页中打开完整的 BIC-QA 界面，提供更大的操作空间和更多功能。适合复杂对话和长时间使用。

基本问答操作

操作步骤

  1. 在首页选择已配置的模型（服务商）
  2. 从知识库下拉菜单选择目标知识库（可选，选择"不使用知识库"则仅使用AI模型）
  3. 选择参数规则（可选，使用默认规则或自定义规则）
  4. 在输入框中输入您的问题
  5. 点击发送按钮或按Enter键提交问题
  6. 等待AI生成回答（支持流式输出，实时显示）
💡 使用技巧：开启知识库后，系统会自动加载最新配置并优先使用知识库服务。

快捷操作

🆕 开启新会话

点击左侧边栏的"开启新会话"按钮，可以开始一次全新的对话，清空当前对话历史。

📚 历史记录

点击"历史记录"按钮，可以查看和管理所有历史对话记录，支持搜索、导出和删除。

📊 AWR分析

点击"AWR分析"按钮，可以上传Oracle AWR报告进行分析，获取专业的性能诊断报告。

❓ 使用说明

点击"使用说明"按钮，可以查看详细的帮助文档和使用指南。

📢 公告

点击"公告"按钮，可以查看最新的产品公告和更新信息。

结果操作

可用操作

  • 📥 导出HTML：将对话内容导出为格式化的HTML文件
  • 📋 复制：快速复制回答内容到剪贴板
  • 🗑️ 清空：清除当前对话内容
  • 👍 点赞：对回答进行点赞反馈，帮助系统优化
  • 👎 否定：对回答进行否定反馈，帮助系统改进
  • ⏹️ 停止生成：在AI生成回答过程中，可以随时点击停止按钮中断生成

多语言支持

BIC-QA 支持多语言界面，您可以在右上角的语言选择器中选择：

  • 中文简体
  • English
  • 日本語

选择语言后，界面会立即切换为对应语言。

🔧 功能详解

智能问答功能

多模型支持

BIC-QA 支持集成多种AI模型，包括：

  • GPT系列（GPT-3.5、GPT-4等）
  • Claude系列
  • 本地Ollama模型
  • 其他兼容OpenAI API格式的模型

流式输出

支持实时流式显示AI回答，无需等待完整回答生成即可看到内容，提升用户体验。

对话管理

自动保存对话历史，支持一键清空、导出对话记录，方便管理和回顾。

知识库检索功能

功能特点

  • 企业集成：无缝连接企业知识库系统
  • 智能匹配：基于语义相似度进行精准内容检索
  • 多库支持：同时管理多个知识库
  • 结果优化：智能排序和相关性过滤

AWR 报告分析功能

功能概述

BIC-QA 提供了强大的 Oracle 数据库 AWR（Automatic Workload Repository）报告分析功能，帮助数据库管理员快速诊断数据库性能问题，获取专业的分析报告和优化建议。

主要功能

  • 智能解析：自动解析 Oracle AWR 报告的 HTML 格式文件
  • 深度分析：基于 AI 大模型进行全面的性能问题诊断
  • 专业报告：生成结构化的分析报告，包含问题诊断和优化建议
  • 多语言支持：支持中文和英文分析报告
  • 历史管理：查看和管理历史分析记录，支持重新分析
⚠️ 版本说明：AWR 报告分析功能当前为 Beta 测试版本。我们诚挚邀请您在使用过程中提供宝贵的意见和建议。

使用步骤

  1. 点击左侧导航栏的"📊 AWR分析"按钮
  2. 在"新建分析"标签页中：
    • 选择数据库类型（Oracle 或 KingBase）
    • 填写系统问题概述（可选，建议填写明确的问题描述）
    • 填写接收邮箱（必填）
    • 上传 AWR 报告 HTML 文件（必填）
    • 选择分析语言（中文或English）
    • 勾选同意免责声明和安全声明
  4. 点击"执行分析"按钮提交任务
  5. 在"历史记录"标签页中查看分析进度和结果
  6. 分析完成后，报告会发送到您填写的邮箱
📋 AWR 报告分析数据源提示：
  • ✅ 支持：Oracle 单实例数据库的 AWR 报告
  • ✅ RAC 建议：如为 RAC 环境，请使用 awrrpt.sql 或 awrrpti.sql 生成报告
  • ❌ 暂不支持：awrddrpi.sql 生成的 AWR 对比报告和 awrgrpt.sql 生成全局报告

分析报告内容

系统生成的 AWR 分析报告通常包含以下内容：

  • 报告概览：数据库版本、报告时间范围等基本信息
  • 负载概况：数据库整体负载情况分析
  • 性能指标：关键性能指标（如命中率、等待事件等）的详细分析
  • 问题诊断：识别出的潜在性能问题和瓶颈
  • 优化建议：针对发现问题的具体优化建议和解决方案
  • SQL 分析：高负载 SQL 语句的分析和优化建议
  • 等待事件分析：等待事件的详细分析和处理建议

历史记录管理

功能特点

🛠️ 故障排除

常见问题

问题 1：插件导入失败

可能原因

  • 解压出现文件损坏
  • 浏览器选择插件时没有选最后一层文件夹

解决方案

  1. 重新下载扩展文件并解压
  2. 确保选择的文件夹下有 manifest.json 文件
  3. 选择最内层的插件文件夹（BIC-QA_v1.1.1），而不是外层压缩包文件夹

问题 2：请求失败

可能原因

  • 模型服务地址错误
  • 网络连接问题
  • API Key 无效
  • 本地模型权限未放行

解决方案

  1. 检查服务地址是否正确：
  2. # 检查本地 Ollama 服务
curl http://localhost:11434/v1/models

# 验证网络连接
ping your-api-endpoint.com
  3. 验证 API Key 是否正确
  4. 对于本地 Ollama，配置系统环境变量：
  5. 设置系统变量 OLLAMA_ORIGINS 值为 *
  6. 检查防火墙设置，确保允许扩展访问

问题 3：模型无法加载

可能原因

  • Ollama 服务未启动
  • API 地址格式错误
  • 模型未正确部署

解决方案

  1. 启动 Ollama 服务：
  2. ollama serve
  3. 检查模型列表：
  4. ollama list
  5. 确认 API 地址格式为：http://localhost:11434/v1
  6. 如果模型不存在，使用 ollama pull <model-name> 下载模型

问题 4：知识库连接失败

可能原因

  • 知识库服务 URL 错误
  • API Key 过期或无效
  • 网络权限限制

解决方案

  1. 检查知识库服务状态
  2. 重新获取 API Key（检查注册邮箱）
  3. 验证网络访问权限
  4. 在设置页面点击"测试连接"验证配置

调试步骤

通用调试方法

  1. 重新加载扩展
    • 打开扩展管理页面
    • 找到 BIC-QA 扩展
    • 点击重新加载按钮
  3. 清除缓存
    • 清除浏览器缓存和 Cookie
    • 重启浏览器
  5. 检查控制台
    • 按 F12 打开开发者工具
    • 切换到 Console 标签页
    • 查看错误信息
  7. 验证配置
    • 检查配置文件格式
    • 确认所有必填字段已填写
    • 验证 API 地址和密钥格式

🔒 隐私安全

数据保护

本地存储

对话历史存储在本地浏览器中，不会上传到服务器。

加密传输

所有网络请求使用 HTTPS 加密传输，确保数据安全。

权限控制

安全特性

  • 本地模型：支持完全本地部署，数据不出本地环境

  • 访问控制：可配置网络访问权限，控制数据流向

隐私政策

  • 不收集个人敏感信息
  • 不向第三方分享数据
  • 支持数据导出和删除
  • 详细隐私政策请查看插件内的"隐私政策"页面
